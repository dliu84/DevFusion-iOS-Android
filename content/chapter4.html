Understanding Object Oriented Programming

Object-Oriented Programming (OOP) is a programming paradigm that provides a means of structuring programs so that properties and behaviors are bundled into individual objects.

In OOP, each object is an instance of a class. The class defines the properties (often referred to as attributes or fields) and methods (actions or functions) common to all objects of that type.

The core principles of OOP include:

  • Encapsulation: The implementation and state of each object are privately held inside a defined boundary, or class.
  • Inheritance: Classes can inherit common state and behavior from other classes.
  • Polymorphism: A principle that allows one interface to represent a general class of actions.
  • Abstraction: The principle of abstraction enables a programmer to hide all but the relevant data about an object to reduce complexity and increase efficiency.

Classes, Structs, and Enums in Swift

Classes: Classes in Swift are general-purpose, flexible constructs that become the building blocks of your code. They include properties (variables and constants) and methods (functions).

Example:


class Dog {
  var name: String
  var breed: String

  init(name: String, breed: String) {
      self.name = name
      self.breed = breed
  }
}

Structs: In Swift, structs are very similar to classes. They can have properties and methods just like classes. However, they are value types whereas classes are reference types. Also, unlike classes, they don't support inheritance.

Example:


struct Point {
  var x: Int
  var y: Int
}

Enums: Enumerations, or more commonly, enums, are a way of defining a type in Swift with a limited set of possible states or values. They are a versatile way to create a group of related values that can be used in your code.

Example:


enum Direction {
  case north
  case south
  case east
  case west
}
