Introduction to Data Persistence in iOS

Data persistence is the mechanism of storing any type of data to disk so that the same data can be retrieved without being altered when the user opens the app next time. It's crucial for any app where you need to save the user's progress, cache data to improve performance, save user settings, or store data offline so your app can be used without a network connection.

In iOS, we have several methods for data persistence, including:

  1. UserDefaults
  2. Files
  3. Core Data
  4. SQLite
  5. Keychain

Using User Defaults for Saving and Loading Data

UserDefaults is an interface to the userâ€™s default database, where you store key-value pairs persistently across launches of your app. It's ideal for storing small pieces of data like user settings and preferences.

Here's how to use UserDefaults to save and load data:

  1. Saving Data:

    You can save data by calling the set(_:forKey:) method on the UserDefaults standard instance. 

    let defaults = UserDefaults.standard
defaults.set("John", forKey: "userName")
  2. Loading Data:

    You can retrieve these values by calling the appropriate method on the UserDefaults standard instance. 

    let defaults = UserDefaults.standard
let userName = defaults.string(forKey: "userName")
  3. Deleting Data:

    To remove a value from UserDefaults, you can use the removeObject(forKey:) method: 

    let defaults = UserDefaults.standard
defaults.removeObject(forKey: "userName")

Please note that UserDefaults is not a secure way to store sensitive data, like user credentials. For that, you should use the Keychain.
